1. Security Threats & Attacks

  • XSS (Cross-Site Scripting): Injection of malicious scripts.
  • SQL Injection: Injecting SQL to manipulate databases.
  • CSRF (Cross-Site Request Forgery): Unauthorized actions on behalf of users.
  • Phishing: Fake sites to steal credentials.
  • Clickjacking: Trick users into clicking hidden elements.

Use input validation, CSP, token-based auth, and secure headers to mitigate.

2. Web Security Model

The web follows a same-origin policy (SOP) that prevents documents from accessing resources from different domains unless CORS headers allow it.

Security relies on layers: HTTPS ‚Üí Auth ‚Üí Access Control ‚Üí Input Validation.

3. Cookies: Set, Access, Delete

4. Anonymous & Authenticated Access

Anonymous Access: Public pages (e.g., home, about).

Authenticated Access: Requires user login. Auth can use:

  • Session Cookies (most common)
  • JWT Tokens (modern APIs)
  • IP Address Restrictions (below)

5. Authentication Types (Simulated)

By IP/Domain: Done server-side. Example:

if (request.ip === "192.168.0.1") { allow(); }

Integrated Windows Authentication (IWA): Uses Windows credentials (for intranet).

Digital Signatures & Certificates: Uses public key infrastructure (PKI).

  • Signature: Confirms origin & integrity
  • Certificate: Issued by trusted CA (like SSL certs)

Demo: HTTPS uses certificates to secure data transfer.

6. Firewalls

Firewall: A system that monitors and controls incoming/outgoing network traffic.

  • Host-based: Runs on devices
  • Network-based: Protects LAN/organization

Used to block ports, IPs, and unauthorized access attempts.
